## Audit Objectives
- Verify that system access is authorized, appropriate, and periodically reviewed
- Ensure access changes are approved and documented
- Confirm evidence exists to support control operation
- Identify gaps that could lead to unauthorized access or compliance risk

---

## Where the Workflow Begins and Ends
**Begins:**  
Audit engagement initiation and scope confirmation.

**Ends:**  
Validated remediation and documented audit closure.

This boundary ensures clear ownership, controlled changes, and a complete audit trail.

---

## Repository Structure
- `/Planning` – Audit planning and risk assessment
- `/Testing` – Control test procedures
- `/Evidence` – Evidence request and tracking
- `/Findings` – Documented audit issues and recommendations

---

## How to Use This Repository
1. Start in `/Planning` to understand scope, risks, and audit approach.
2. Use `/Testing` to execute control tests and document results.
3. Track artifacts and requests in `/Evidence`.
4. Document issues and recommendations in `/Findings`.
